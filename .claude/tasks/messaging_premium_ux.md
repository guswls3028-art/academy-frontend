# 메시지 도메인 프리미엄 UX 작업 명세

## 목표
대기업 프로그램 수준의 메시지 발송 UX. 사용자가 이질감 없이 바로 사용 가능.

## 현재 상태 (2026-04-02)
- SMS 발송: 정상 동작 (뿌리오 연동, 실발송 확인)
- 알림톡: 드롭다운 선택 + readOnly 미리보기 동작
- 성적 양식: 자동 로드 + 변수 치환 동작
- 설정 페이지: KPI 4개 표시

## 남은 작업

### 1. 메시지 도메인 탭 레이아웃 통일
**현재**: 탭이 페이지 헤더 내부 (다른 도메인과 다름)
**목표**: 다른 도메인(학생, 강의 등)과 동일한 DomainLayout > Tabs 구조

작업:
- MessageLayout.tsx에서 도메인 레이아웃 탭으로 변경
- 탭: 템플릿 저장 | 자동발송 | 발송 내역 | 설정
- 사이드바 "메시지" 클릭 시 첫 탭(템플릿 저장)으로 이동
- routes.tsx 수정

### 2. 알림톡 전체 ON/OFF 토글
**현재**: 개별 자동발송에서만 끄기 가능
**목표**: 설정 페이지에 "알림톡 사용" 마스터 토글

작업:
- 설정 페이지 카카오 알림톡 섹션에 마스터 토글 추가
- 끄면: 알림톡 자동발송 전체 비활성, 모달에서 알림톡 탭 숨김
- 백엔드: tenant.alimtalk_enabled 필드 또는 기존 필드 활용

### 3. 발송 모달 프리미엄 UX
**목표**: 정보 계층 명확, 운영자 실수 방지

개선:
- 대상 요약 카드: 총 대상 수, 발송 가능/불가 수, SMS/알림톡 가능 수
- 번호 없는 대상 분리 표시 (발송 불가 사유)
- 변수 블록 패널: 변수명(라벨) + 실제 값 + 누락 여부
- 템플릿 변경 시 프리뷰 즉시 재계산 확인
- 스크롤 없이 핵심 정보가 한눈에 보이게

### 4. 성적 발송 양식 완성도
**목표**: 림글리시 양식 100% 재현

개선:
- 시험 데이터 없는 학생: 빈 양식이 아닌 명확한 안내
- 다건 선택 시: 개별 양식 대신 "N명에게 동일 양식으로 발송" UX
- 양식 하단(과제 안내) 편집 영역 명확 분리
- 변수 블록에서 시험명/만점이 실제 데이터로 미리보기

### 5. 숨은 버그 탐색 체크리스트
- [ ] 학부모만 번호 있는 학생에게 발송 시 동작
- [ ] 학생만 번호 있는 학생에게 발송 시 동작
- [ ] 둘 다 번호 없는 학생 선택 시 발송 버튼 상태
- [ ] 90자 초과(LMS) 발송 시 요금 안내
- [ ] 2000자 초과 시 차단
- [ ] 알림톡 미승인 템플릿 선택 시 발송 차단
- [ ] 빈 본문으로 발송 시 차단
- [ ] 학생/학부모 토글 전부 해제 시 발송 버튼 상태
- [ ] 템플릿 저장 후 바로 불러오기 가능한지
- [ ] 모달 닫기 → 재열기 시 이전 입력 초기화
- [ ] 대량 선택(50명) 시 모달 성능
- [ ] 성적 양식에서 시험 0건일 때 빈 줄 처리

## 실행 순서
1. 메시지 탭 레이아웃 통일
2. 발송 모달 프리미엄 UX
3. 알림톡 ON/OFF 토글
4. 성적 양식 완성도
5. 숨은 버그 전수 체크
6. 전 도메인 E2E
7. 실발송 검증
8. 배포 + 사후 검증

## 설계 원칙
- 기존 동작을 깨지 않는다
- 추가 기능보다 기존 기능의 안정성 우선
- 폴백/암묵 분기 금지
- 프리뷰와 실발송 SSOT 유지
- 운영 계정 비밀번호 변경 절대 금지
